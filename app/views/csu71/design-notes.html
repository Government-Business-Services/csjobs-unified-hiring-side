V2


Aim

V2 was an iteration of the initial V1 prototype. Based on adding multi-grade / contract functionality and some improvements based on user feedback from V1, our aim was to:

  • allow a user to build a single ad with multiple job types e.g. different grades or contract types
  • increase contact options by offering a 'team mailbox' option
  • amending content where issues had been raised during V1 testing

What we did

We made the above changes to create a V2 version of 'create a vacancy'.

V1


Aim

Our aim was to:

  • offer a user the ability to create a bulk campaign within the new service
  • allow a user to create a rolling or tranche campaign

What we did

We amended the existing 'create a vacancy' prototype and added in functionality for a bulk or rolling campaign, including capturing number of posts, what type of bulk campaign it is etc.

  • V1 'create a vacancy' for bulk campaigns

User research

Mural board with testing notes


What we found

  • some users need to create bulk campaigns with more than one 'job type' in the ad: for example, multiple grades or different contract lengths / types
  • choosing a campaign duration (e.g. 2, 3 or 4 weeks) did not work for many users - they preferred the ability to add a campaign end date instead
  • many users would not want to choose a single person or two people as contacts - instead they wanted the option to use a team / generic email address to capture queries
  • adding more questions related to vacancy numbers did not make the 'Basic details' page overwhelming for users
  • online tests needed more promotion / explanation of benefits to encourage users to opt for them as a sift method
